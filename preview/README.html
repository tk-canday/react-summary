組件的生命周期

  1. componentWillMount() 组件挂载之前（建议在这挂载异步操作）

  2. componentWillReceiveProps(nextProps) props即将更新。(可以在此时更新state以响应prop变化）

    父组件发生render的时候子组件就会触发此事件 示例：

    componentWillReceiveProps(nextProps) {
if (nextProps.data.count) {
 this.setState({
   count : this.props.data.count
 })
}
}

  3. componentDidMount() 组件挂载完成，一个生命周期内只会调用一次(在此处注册监听事件)

  4. shouldComponentUpdate（） 每次setState都会调用此事件判断是否需要从新渲染，默认返回true。

    有两个参数(nextProps ，nextState)

  5. componentWillUpdate() 组件更新之前 (可以监视state变化做一些操作)

    当 shouldComponentUpdate 返回 true 或者调用 forceUpdate 后被调用

  6. componentDidUpdate() 除了首次render后调用componentDidMount()，其他render 结束之后都是调用此方法

  7. componentWillUnmount() 即將銷毀(在此处一定要移除注册的事件)

render

  • this.isMounted() 如果组件已经渲染到了DOM上，返回true

  • this.forceUpdate（） 强制更新组件

    当组件中绑定的数据不在 state 或者 props 中时候，或者未用 setState更新state，

    组件无法自动更新，就需要我们手动进行更新

  • 触发render的行为：
    • 组件的首次渲染
    • 调用 this.setState（）
    • 父组件发生更新（不管props有没有发生变化）
    • 调用 this.forceUpdate（）

动态渲染子组件

key属性: react算法效率问题,且如果没有这个属性的话不能保证渲染的顺序

应当把key添加到子集数组的子组件本身上，且为了排序正确最好加一个字符串前缀

注意：key不是props，不能当作props进行传递

  1. 在父亲组件中循环给子组件设置属性 or 子组件中设置属性...
  2. 将子组件存入数组 or 子组件接受父组件的数据
  3. 在父组件中，遍历子组件数组循环渲染子组件

示例：

    this.props.po(this.num)

  constructor (...args) {
    super(...args)
      this.state = {arr:['item1','item2','item3']}

React 混入机制

mixins @ES5

虽然组件的原则就是模块化，彼此之间相互独立，但是有时候不同的组件之间可能会共用一些功能，共享一部分代码。所以 React 提供了 mixins 这种方式来处理这种问题。

Mixin 就是用来定义一些方法，使用这个 mixin 的组件能够自由的使用这些方法（就像在组件中定义的一样），所以 mixin 相当于组件的一个扩展，在 mixin 中也能定义“生命周期”方法，而且组件会先执行自己定义的方法再依次执行混入的方法。

var DefaultNameMixin = {
    getDefaultProps: function () {
        return {name: "TOM"};
    }
};

var DefaultFoodMixin = {
    getDefaultProps: function () {
        return {food: "Noodles"};
    }
};
var ComponentTwo = React.createClass({
    mixins: [DefaultNameMixin, DefaultFoodMixin],
    render: function () {
        return (
            <div>
                <h4>{this.props.name}</h4>
                <p>Favorite food: {this.props.food}</p>
            </div>
        );
    }
});

注意：

  1. 不同的mixins中不能定义相同的 props 与state
  2. 在不同的mixin中定义相同的方法，或者mixin和组件中包含了相同的方法时，会抛出异常
  3. 如果组件和 mixins 中有相同的周期函数，调用顺序为：组件中定义的周期函数 ==> mixins 数组中的顺序
  4. ES6编写的组件不支持此语法
